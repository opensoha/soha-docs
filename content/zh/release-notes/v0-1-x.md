# OpenSoha 0.1.x Beta 发布说明草案

本文是 `0.1.x` beta release notes 草案。它只记录当前本地和 CI 可验证证据；真实 tag、GitHub release、npm publish、image publish 和公网 smoke 完成后必须补充远端 URL。

## Release Scope

- `soha-contracts`: OpenAPI、JSON Schema、Go/TypeScript SDK、release manifest、consumer matrix。
- `soha-web`: Vite dist tarball、checksum、release artifact verifier。
- `soha`: core binary、embedded Web artifact、pinned contracts module、release manifest。
- `soha-cli`: standalone release binary、checksum、`soha add` install smoke。
- `soha-agent`: release binaries、GHCR image、checksum、post-publish asset verification。
- `soha-skills`: installable skills package、manifest、validation report。
- `soha-connectors`: connector package, manifest registry, Feishu runtime, contracts-owned event envelope mirror.
- private extension slices: kept outside the public release artifact boundary.

## Upgrade Notes

- Upgrade contracts consumers only after `@opensoha/contracts` is published and the Go module tag resolves.
- Core release inputs must pin `contracts_ref`, `web_ref`, and optional `web_sha256`.
- Web release tarball and checksum must be downloaded and verified before core embed.
- CLI and Agent installation smoke must use release binaries, not sibling source checkouts.
- Public-release dependency mode remains pending until `@opensoha/contracts` exists in npm.

## Rollback Notes

- Core rollback must restore the previous binary, release manifest, contracts module version, and embedded Web checksum together.
- Web rollback must restore the previous dist tarball and `.sha256` pair.
- Contracts rollback cannot unpublish public npm safely; publish a forward fix and keep compatibility notes.
- CLI/Agent rollback should pin previous release binaries and checksums in install docs.
- Skills/Connectors rollback should keep previous package manifests and validation reports available as release assets.

## Verification Evidence

Local evidence currently covers unit, type, package, artifact, checksum, and docs consistency gates. Pending real evidence:

- `@opensoha/contracts@0.1.0` npm publish.
- `soha-contracts` GitHub release assets.
- `soha-web` and `soha` real release tags and assets.
- CLI and Agent release binary install smoke on clean machines.
- Docs publish workflow success and public `docs.opensoha.dev` smoke.

## Known Gaps

- Full cold-start smoke still needs real Postgres, Docker image, login, authenticated API, embedded Web, and Helm production values in one chain.
- Product e2e smoke still needs login, permission filtering, cluster scope, K8s read, delivery, AI Gateway, Docker runtime, and Agent callback.
- Feishu sandbox, public contracts cutover, external image/chart publish, provenance/SBOM/signing, and ecosystem rollback governance remain pending.
